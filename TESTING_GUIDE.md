================================================================================
ALGORITHM EXPLANATIONS
================================================================================

1. # Q-LEARNING
   Type: Off-Policy Tabular Method
   Concept: Learns the optimal policy while exploring
   Update Rule: Q(s,a) ← Q(s,a) + α[r + γ·max(Q(s',a')) - Q(s,a)]

Characteristics:
✓ Learns the BEST possible policy
✓ Can learn from experiences generated by any exploratory policy
✓ Faster convergence in many tasks
✗ May be unstable if exploring too much
✗ Requires discrete state/action spaces (tabular method)

2. # SARSA
   Type: On-Policy Tabular Method  
   Concept: Learns the value of its current exploratory policy
   Update Rule: Q(s,a) ← Q(s,a) + α[r + γ·Q(s',a') - Q(s,a)]

Key Difference: Uses actual next action a', not max action

Characteristics:
✓ More conservative approach (accounts for exploration)
✓ Better for environments with dangerous states
✓ More stable learning
✗ Learns a mediocre policy (includes exploration noise)
✗ Slower convergence
✗ Requires discrete state/action spaces (tabular method)

3. # DQN (Deep Q-Network)
   Type: Off-Policy Deep Learning Method
   Concept: Uses neural network to approximate Q-values instead of lookup table
   Architecture: Input(125) → Hidden(64) → Hidden(64) → Output(6)

Key Features:
• Experience Replay: Learn from mini-batches of stored transitions
• Target Network: Separate network to stabilize training
• Neural Network: Can handle high-dimensional feature vectors

Characteristics:
✓ Handles high-dimensional state spaces (vision, etc.)
✓ Generalizes across similar states
✓ More flexible and powerful
✗ Requires feature engineering
✗ More training data needed
✗ Slower per-episode training
✗ More hyperparameters to tune

EXPECTED PERFORMANCE
================================================================================

The cleaning task requires the robot to:

1. Explore the 18×12 grid environment
2. Learn which tiles are dirty
3. Clean all dirty tiles by moving and using clean action
4. Return to charging station (8, 9) for bonus reward

Success Criteria:
• All 108 cleanable tiles must be cleaned (DIRTY → CLEAN)
• Robot must return to charging station (8, 9)
• Reward for successful completion: +300 (100 for all clean + 200 for at charger)

Performance Metrics Tracked:

1. Episode Reward: Total reward accumulated in each episode
2. Success Rate: Percentage of episodes that complete successfully
3. Episode Length: Number of steps needed to complete (shorter is better)
4. Epsilon: Exploration rate (starting at 1.0, decaying to ~0.02)

Typical Learning Curves:
• Reward: Starts negative (exploration), increases as agent learns
• Success: Starts at 0%, gradually increases
• Episode Length: Starts high (random exploration), decreases (learning)

2. SUCCESS RATE COMPARISON

   What to look for:

   • Plateau at high % = mastered the task

3. CUMULATIVE SUCCESSES
   X-axis: Episode number
   Y-axis: Total number of successful episodes so far

   What to look for:
   • Steep slope = high success rate currently
   • Steeper curves are better
   • Final count = total successes

   Expected: Q-Learning > SARSA > DQN (in that order typically)

4. EPISODE LENGTH
   X-axis: Episode number
   Y-axis: Number of steps in episode

   What to look for:
   • Downward trend = agent learning efficient paths
   • Plateau at low value = learned optimal behavior
   • Lower plateaus = more efficient policies
